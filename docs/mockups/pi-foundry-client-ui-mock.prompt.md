Create a high-fidelity product UI concept board for a Foundry-first pi-based agent client.

Product framing:
- This is not a generic chatbot.
- It is a user-first task workspace for long-running agents, with a secondary developer studio for building pi-based agents.
- It maximally reuses Microsoft Foundry for hosting, deployment, versioning, identity, and runtime control.
- The product layer focuses on task execution, artifacts, agent catalog, agent studio, releases, Foundry bindings, and run history.

Audience:
- Primary: end users who want an agent to finish real work.
- Secondary: developers who build pi-based agents using skills, MCP, tools, and environment profiles.

Show a multi-screen desktop concept board with 6 clear app screens:
1. Home / task intake screen with a large prompt box saying “What do you want to get done?”, recommended agents, recent tasks, and category chips.
2. Agent detail screen with capability summary, expected outputs, permissions, long-running support, and a launch button.
3. Task workspace screen with streaming response in the center, progress timeline on the right, artifact panel, session status, and background-run indicator.
4. Tasks screen showing queued, running, waiting-for-user, completed, and failed runs with status chips and durations.
5. Artifact gallery / preview screen showing HTML report, audio narration, zip bundle, and preview panel.
6. Builder Studio screen showing Agent Definition, skills, MCP, tools, env profiles, releases, and Foundry deployment binding.

Information architecture details:
- Left navigation with Home, Agents, Tasks, Artifacts, History, Studio.
- A premium but practical enterprise/productivity style.
- Distinguish end-user surfaces from developer surfaces using subtle labeling, not heavy visual separation.
- Include cards or chips for: AgentDefinition, Release, Run, Artifact, Foundry Binding.
- Include visible long-running concepts such as resume, background run, reconnect, and artifacts.
- Include a small release/deploy section with Foundry version and healthy endpoint badge.

Visual style:
- Clean SaaS dashboard aesthetic.
- Soft dark-neutral canvas with luminous blue-violet accents.
- Modern enterprise UI, Apple/Linear/Vercel inspired restraint.
- Sharp card layout, large readable typography, clear hierarchy, minimal clutter.
- Use realistic interface details, not sci-fi holograms.
- Figma-style board presentation, slightly angled or top-down is okay, but keep screens readable.

Constraints:
- No watermark.
- No giant fake paragraphs of nonsense text.
- No stock-photo people.
- No browser chrome unless necessary.
- Make the UI feel buildable.
- Use concise realistic labels and legible structure.
